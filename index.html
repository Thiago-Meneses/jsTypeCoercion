Type Coercion

Type Coercion can be described as the automatic or implicit conversion of values from one data type to another.

This characteristic makes JavaScript a very forgiving language when it comes to types.

To better understand what Type Coercion actually means, it's important to learn the meaning of the word Coerce:

Definition of Coerce:

-To Persuade or Obtain something by using force.

Coming back to programming, Type Coercion is basically telling us that JavaScript will force a certain type of value to be converted into another type when we are doing operations.

Some operators in will specifically convert values into numbers.

Operators:

  • -
  • *
  • /
  • %

Truthiness

JavaScript is able to convert countless types of values into true or false

Some values are truthy, because they can get converted into true.

Some values are falsy, because they can get converted into false.

Falsy values

  • false
  • 0n
  • 0
  • -0
  • ""
  • NaN
  • null
  • undefined

Truthy values

  • All values that aren't falsey
  • Because of this, most values in JavaScript end up beng converted into true.
